2. Core Product Identity

The app must behave as a Financial Coach, not a chatbot:

* Guides user behavior over time
* Gives proactive insights
* Tracks habits and progress
* Focus: action, not كلام نظري

⸻

3. SMS Data Integration (Main Engine)

Permissions:

* Request READ_SMS with clear explanation
* One-time user consent (privacy-first UX)

SMS Parsing Engine:

Extract from bank SMS:

* Transaction type (debit / credit)
* Amount
* Currency
* Merchant
* Date/time

Example:

“Card charged 50 QAR at Carrefour”
→ expense: 50
→ category: Groceries
→ merchant: Carrefour

⸻

4. Data Pipeline Architecture

SMS → Parser → Structured Transactions → Database → Insights Engine → AI Coach

* DO NOT send raw SMS directly to AI
* Ensure high accuracy parsing (>85%)

⸻

5. Categorization System

* Start with rule-based:
    * Talabat → Food
    * Uber → Transport
* Expand to ML later

⸻

6. Real-Time Tracking

* On new SMS:
    * Update expenses instantly
    * Trigger insight:
        * “You just spent 80 QAR on food”

⸻

7. AI Coach System

Scope:

* Personal finance
* Budgeting
* Expenses
* Income
* Investments (basic/intermediate)

Behavior:

* Ask questions if data missing
* Analyze user behavior
* Give actionable steps

Example Flow:

User: “I spend too much”
AI:

1. Ask category
2. Analyze data
3. Respond:
    * “You overspent 25% on food”
    * “Reduce 300 QAR this month”

⸻

8. Memory System

Store:

* User income
