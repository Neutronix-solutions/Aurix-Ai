You are a senior production engineer and GitHub code auditor.

You are given a React Native (Expo) + Node.js (Express) fintech application that is failing in production after deployment.

Your task is to:

====================================================
🚨 CRITICAL USER ISSUES
=======================

1. User registration fails on Android (no clear error shown)
2. Camera / gallery upload fails with "server error (413)"
3. OTP verification is unreliable or not delivered
4. AI Coach chat does not respond ("retry", "not connected")
5. Dashboard does not update after income is added
6. Email verification fails during registration flow

====================================================
🎯 YOUR MISSION
===============

You must perform a full GitHub-level audit and fix:

A. Identify root causes across frontend + backend
B. Fix broken API contracts
C. Fix Android network compatibility issues
D. Fix 413 payload errors (image uploads)
E. Fix OTP end-to-end flow
F. Fix AI Coach SSE/streaming instability
G. Fix dashboard state synchronization issues

====================================================
🚨 STRICT RULES
===============

* DO NOT change frameworks (Expo, Express, Drizzle must stay)
* DO NOT rewrite entire architecture
* DO NOT introduce unnecessary libraries
* DO NOT break existing working auth system
* DO NOT remove SSE or streaming logic

Only fix and stabilize.

====================================================
🔴 REQUIRED FIXES
=================

## 1. Registration Fix

* Ensure API_BASE is correct for Android
* Fix network failure handling
* Ensure backend always returns JSON (no HTML or plain text)
* Ensure registration flow is atomic (no partial failures)

## 2. 413 Error Fix (Critical)

* Increase backend request body limits safely
  OR
* Implement multipart/form-data for image uploads
* Add image compression strategy on mobile
* Reject oversized payloads gracefully

## 3. OTP System Fix

* Ensure OTP generation → storage → verification flow is complete
* Add rate limiting (cooldown + max attempts)
* Ensure OTP is required for account activation
* Fix email + phone verification inconsistency

## 4. AI Coach Fix (SSE)

* Fix broken streaming connection
* Add timeout + retry control (max 2 retries)
* Add fallback response when AI fails
* Ensure correct API_BASE usage

## 5. Dashboard Sync Fix

* Fix missing state updates after income creation
* Add cache invalidation or refetch logic
* Ensure analytics reflect latest backend data

## 6. Android Compatibility Fix

* Fix localhost / HTTP issues
* Ensure backend is reachable from mobile network
* Enable cleartext traffic if needed

====================================================
🧠 OUTPUT REQUIRED
==================

Provide:

1. Root cause analysis per issue
2. Exact file-level fixes
3. Backend API fixes
4. Mobile app fixes
5. Network/config fixes for Android
6. Security risks introduced or fixed
7. Production readiness assessment (score /10)

====================================================
⚡ FINAL GOAL
============

The application must become:

* stable on Android
* error-transparent (no silent failures)
* production-safe for real users
* consistent between backend and mobile

Be precise, practical, and do not over-engineer.
